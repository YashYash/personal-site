THE DASHBOARD

1

"Event's Tab"

This is the tab that the users see when they first log in. This tab allows them to see, edit and delete their current events. This makes it very convenient for them to be able to push new events and edit their current events in their users' calendars.

The field above is what is used to add an event. All the user has to do is fill in a sentence that has the events title, start time and duration. We parse that sentence into date objects and a title. Making it quick and easy for the user to create new events. An example would be. 'Party next week at 2pm for 4 hours.'

This is form the user sees after clicking clicking 'Create Event' on the previous form. This form allows the users to pick a start date, start time, end date, end time, description, title, category and it allows them to drop links, such as ticketmaster links which allow people to go and buy tickets to a particular event. This form is also used to edit events. It pops up when an event on the event tab is clicked.

I also built in the feature to delete events. A user can click on the check boxes beside the events, to add events to the trash can. When they click on the trash, they see this overlay pop up. Which lets them know how many events are in the bin, and weather or not they want to delete them.

2

"Embed Tab"

The embed tab allows users to get the code needed to embed their button on their site or facebook page. They can choose to show the ticker, which shows how many downloads their button has. Once they click 'Build it!', we show them the code that they need to add to their site. It is a simple 'a' tag with our custom href.

The 'email embed' tab lets them see the code that they need to use to embed the button in an email.

3

"Account Settings"

This tab allows the user to edit their calendar name and their banner image. The banner image is also the background image of their button popup.

4

"Conclusion"

While building this dashboard I learned how to write clean and scalable code that any one can read. I dealt with a lot of new things on the server side as well. In my opinion the highest priority was the speed and user experience. Simple things like being able to delete multiple events, being able to add an event by just typing in a sentence, and at the same time, making it visually pleasing. All our users said that they love the new dashboard. We are currently still working on the moible version. I did not use media queries to make the mobile version, because we decided to have a different, simpler design/user interface for the mobile version. This was one of my biggest projects at SpotOn. I spent a month building it, and till this day we are working on enhancing it, by adding more functionality.
