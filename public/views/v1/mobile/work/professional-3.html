THE FLOW

1

"Click it!"

This is our button on the Stanford website. We have no say on how they choose to place it on their site. We provide them with a script tag and an a tag. They can place it where ever they want. (They usually end up picking the worst spots...)

2

"Choose a Category!"

Once the button is clicked, the popup opens. You can see the mobile and web view of this above. On this screen, the user is asked to choose a category that they would like to download.

3

"Choose a Calendar!"

After a category has been selected, the user is asked to choose what calendar they would like to download it to. They will be asked to authenticate if they choose google or exchange. ICAL directly adds the events to your mobile calendar. The desktop version has big tiles that have a transparency hover effect which lowers their opacity with a suttle 0.3s transition. The user can click the back button in the navbar, if they choose the wrong category. Whenever you switch routes, the page does not refresh, only the states change, giving it a native feel.

4

"That's it!"

And that't it! Once a calendar is chosen, the events are automatically added. The user is shown a confirmation screen letting them know that everything went as planned. We also show them recomendations, and allow them to share this category on social media.

Conclusion

I was glad that I got the opportunity to build this button popup. When I built this, I built two versions for AB testing. Having done this I learned how to implement AB testing using UI router and sub routes. This popup has been tested on every browser, ever screen size and on both IOS and Android devices. It works consitently. It is nice to have built this, as it is being used by millions of people.
